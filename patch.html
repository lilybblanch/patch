    The brief

    What you're building

    A friendly "threat companion" that lives in front of your clicks. Safe links feel normal — the cursor even changes shape like it always does. But when you drift toward something risky, Patch physically gets between you and it, then offers three honest options: inspect why, run it anyway, or high-five him for the save. It's part safety tool, part personality.

    Level · beginner-friendly ~1 hour Skills · custom cursors · vector math · character animation AI · swap the rules for a real verdict

    Name the parts before you animate them

    Custom cursor
    We hide the OS cursor and draw our own dot + ring, so we control exactly where it can go.
    Danger zones
    An invisible radius around each risky file. Cross it and the guard kicks in.
    Patch
    A faceless blob with two arms. All his feeling comes from posture — idle, shove, high-five.
    The push
    A tiny bit of vector maths that keeps the cursor clamped outside the danger radius.
    The choice card
    Three real buttons, parked in safe space so you can always reach them.

    Step 1

    Take over the cursor (but keep what people expect)

    Hide the real cursor with cursor:none, then draw your own and glide it toward the mouse each frame with a little easing (a "lerp"). The lag is what makes it feel soft. Lily's note: keep the familiar behaviour — over a safe link the ring still swells, so it reads as "this is clickable," exactly like a normal pointer.

    .screen { cursor: none; }   /* hide the OS arrow */

let tx=0, ty=0, cx=0, cy=0;        /* target vs rendered */
screen.addEventListener('mousemove', e => {
  const r = screen.getBoundingClientRect();
  tx = e.clientX - r.left;  ty = e.clientY - r.top;
});
function loop(){
  cx += (tx - cx) * 0.22;          /* ease toward the mouse */
  cy += (ty - cy) * 0.22;
  /* …push logic goes here (Step 3)… */
  cursor.style.transform = `translate(${cx}px,${cy}px)`;
  requestAnimationFrame(loop);
}
loop();
    Step 2

    Mark what's safe, and what isn't

    Each file is just a tile with a data-safe flag. The risky ones carry an id we can look up later for their red flags. Right now we decide safety by hand — in a moment we'll hand that judgement to an AI.

    <div class="tile" data-safe="1"> Q4-report.pdf </div>
<div class="tile bad" data-safe="0" data-id="iphone"> FREE_iPhone.exe </div>

const RADIUS = 108;   /* the size of each danger zone, in px */
    Step 3

    The push — a pinch of vector maths

    Every frame, measure from the cursor to each risky tile's centre. If you're inside the radius, don't let the cursor in: snap it back to the edge of the circle, along the line it came from. The cursor slides around the rim instead of reaching the file — which reads, beautifully, as being gently shoved away.

    for (const tile of riskyTiles) {
  const c  = centreOf(tile);
  const dx = cx - c.x, dy = cy - c.y;
  const dist = Math.hypot(dx, dy);

  if (dist < RADIUS && !allowed[tile.id]) {
    const ux = dx / dist, uy = dy / dist;   /* direction out */
    cx = c.x + ux * RADIUS;                  /* clamp to the rim */
    cy = c.y + uy * RADIUS;
    summonPatch(c, ux, uy);                  /* stand him in the gap */
  }
}
    Step 4

    Build Patch — all feeling, no face

    Patch is a few rounded <div>s: a body, a head, a little shield, and two arms pinned at the shoulder. There's no face on purpose — every emotion comes from posture. Swapping a class rotates his arms: down for idle, thrust out to shove, one straight up for a high-five. That's the whole trick.

    .arm          { transform-origin: 50% 8px; }   /* pivot at shoulder */
.arm.l        { transform: rotate(28deg); }    /* resting */
.alert .arm.l { transform: rotate(118deg); }   /* SHOVE */
.hi .arm.r    { transform: rotate(-165deg); }  /* up high! */

.arm { transition: transform .45s cubic-bezier(.5,-.3,.3,1.4); }
    Step 5

    The choice — inspect, run, or high-five

    The card lives at the bottom, well clear of the danger zone, so you can always click it. Each button has a clear job, and none of them lie to the user:

    01

    Inspect

    Slides open the actual red flags for that file — a look-alike domain, an .exe from the web, urgent wording. It teaches, instead of just blocking.

    02

    Run anyway

    Respects the human. Patch steps aside for a few seconds and lets you through — because a guard that can't be overruled is just a wall.

    03

    High-five

    The reward path. Patch throws an arm up, your cursor flies to his mitt, sparks pop. Pure delight — and the reason people will actually keep him around.

    ⌥ accessibility

    The buttons are real, focusable buttons and the whole thing still works from the keyboard. Honour prefers-reduced-motion so the shove becomes a calm fade. Fun should never lock anyone out.

    Step 6 — the AI Workspace bit

    Make the verdict real with AI

    In the demo, "is this risky?" is a hand-written list. In the real world that's exactly the job you hand to AI. This is where the project graduates from a fun front-end into something that belongs in the Automate & Secure phase of the AI Workspace roadmap — right next to the AI Security Scanner.

    You build the whole thing inside your AI workspace: let Cursor or Claude Code scaffold the boilerplate while you direct the craft, then replace the rules with a single call that asks an AI to judge a link or file and explain itself.

    // the production swap: ask AI for a verdict + reasons
async function getVerdict(file) {
  const res = await fetch('/api/scan', {
    method: 'POST',
    body: JSON.stringify({ name: file.name, url: file.url })
  });
  return await res.json();   // → { risk: 0.92, reasons: [ … ] }
}

// Patch just reacts to the score — UI stays exactly the same
if (verdict.risk > 0.6) guard(file, verdict.reasons);
    Phase 2 · AI Workspace
    Where it slots in. The front-end teaches transforms & interaction; the AI verdict teaches calling a model and acting on its output; and pulling live signals (domain age, reputation) through MCP connects it to real infrastructure — the same three skills the roadmap already builds, wrapped in something a learner genuinely wants to make.
    honest by design

    Patch never silently blocks and never pretends to be certain — he shows the risk score, lists his reasons, and hands the final call back to you. That's the right posture for any AI safety tool: assistive, transparent, overridable.
